## 2. 전환 포인트 설계

- **비회원 → 무료**: 저장하려는 순간 막힘

- **→ 교사**: 클래스룸 필요 시

## 4. 구현 로직

### 4-1. Firestore 유저 문서 추가 필드

```
users/{uid}:
  plan: 'free' | 'standard' | 'premium' | 'teacher'
  planExpiry: timestamp          // 유료 플랜 만료일
  usage: {
    summarize: 0                 // 이번 달 요약 횟수
    summarizeToday: 0            // 오늘 요약 횟수
    summarizeTodayDate: 'YYYY-MM-DD'
    docent: 0
    blog: 0
    shorts: 0
    quiz: 0
    itinerary: 0
    multiReport: 0
    resetAt: '2026-05-01'        // 다음 초기화 날짜 (매월 1일)
  }
```

### 4-2. 비회원 추적 방식

- 브라우저 `localStorage`에 요약 횟수 저장
- `{ guestSummarizeToday: 1, guestDate: 'YYYY-MM-DD' }` 형태
- 날짜 바뀌면 자동 초기화 (하루 2개 리셋)
- 2회 초과 시 회원가입 유도 모달 표시

### 4-3. 플랜 한도 상수 (`lib/plans.ts`)

```typescript
export const PLAN_LIMITS = {
  guest:    { summarizeDay: 2,  summarize: -1, docent: 0,  blog: 0,  shorts: 0,  quiz: 0,  itinerary: 0,  multiReport: 0,  folders: 0  },
  free:     { summarizeDay: 3,  summarize: 30, docent: 10, blog: 3,  shorts: 3,  quiz: 0,  itinerary: 0,  multiReport: 0,  folders: 3  },
  standard: { summarizeDay: 10, summarize: 100,docent: -1, blog: 20, shorts: 20, quiz: 30, itinerary: 10, multiReport: 5,  folders: 10 },
  premium:  { summarizeDay: 30, summarize: -1, docent: -1, blog: -1, shorts: -1, quiz: -1, itinerary: -1, multiReport: -1, folders: -1 },
  teacher:  { summarizeDay: -1, summarize: -1, docent: -1, blog: -1, shorts: -1, quiz: -1, itinerary: -1, multiReport: -1, folders: -1 },
  // -1 = 무제한
}
```

### 4-4. API 체크 흐름

```
요청 수신
  ↓
유저 식별 (Firebase ID Token or 비회원)
  ↓
플랜 + 사용량 조회
  ↓
resetAt < 오늘? → usage 월 카운터 초기화, resetAt = 다음 달 1일
summarizeTodayDate < 오늘? → summarizeToday = 0, 날짜 갱신
  ↓
일 한도 또는 월 한도 초과?
  → YES: 402 응답 + 업그레이드 안내 메시지 반환
  → NO:  API 처리 실행
          ↓
         Firestore usage 해당 항목 +1 (원자적 increment)
          ↓
         결과 반환
```

## 5. 결제 연동 (🔲 미확정)

### 후보
- **토스페이먼츠** — 국내 1위, 정기결제 지원
- **포트원(아임포트)** — 카카오페이·네이버페이 통합

### 결제 흐름
```
유저 플랜 선택
  → 결제 위젯 오픈 (토스페이먼츠 SDK)
  → 결제 완료
  → 결제 웹훅 수신 (/api/webhook/payment)
  → Firestore users/{uid} 업데이트:
      plan = 'standard' | 'premium' | 'teacher'
      planExpiry = 오늘 + 30일
  → 유저에게 성공 화면 표시
```

### 정기결제 (구독)
- 토스페이먼츠 자동결제(빌링키) 사용
- 매월 planExpiry 전날 자동 청구
- 실패 시 3일간 재시도 → 실패하면 free로 다운그레이드

## 6. 프론트엔드 처리

### 한도 초과 시 UX
- API에서 `{ error: 'LIMIT_EXCEEDED', plan: 'free', feature: 'summarize' }` 반환
- 클라이언트에서 **업그레이드 모달** 자동 표시
- "오늘 요약 3회를 모두 사용했어요. 스탠다드로 업그레이드하면 하루 10회 이용 가능합니다."

### 사용량 표시
- 마이페이지 프로필 카드에 이번 달 사용량 바 표시
- ex) `요약 12 / 30회`
