## 3. 구현 로직

### 3-1. Firestore 유저 문서 추가 필드

```
users/{uid}:
  plan: 'free' | 'standard' | 'premium' | 'teacher'
  planExpiry: timestamp          // 유료 플랜 만료일
  usage: {
    summarize: 0                 // 이번 달 요약 횟수
    docent: 0
    blog: 0
    shorts: 0
    quiz: 0
    worksheet: 0
    itinerary: 0
    multiReport: 0
    resetAt: '2026-05-01'        // 다음 초기화 날짜 (매월 1일)
  }
```

### 3-2. 비회원 추적 방식

- 브라우저 `localStorage`에 요약 횟수 저장
- `{ guestSummarizeCount: 2, lastUsed: '2026-04-17' }` 형태
- 3회 초과 시 회원가입 유도 모달 표시
- (한계: 브라우저 초기화 시 리셋됨 — 의도적으로 허용)

### 3-3. API 체크 흐름

```
요청 수신
  ↓
유저 식별 (Firebase ID Token or 비회원)
  ↓
플랜 + 사용량 조회
  ↓
resetAt < 오늘? → usage 전체 0으로 초기화, resetAt = 다음 달 1일
  ↓
이번 달 사용량 >= 플랜 한도?
  → YES: 402 응답 + 업그레이드 안내 메시지 반환
  → NO:  API 처리 실행
          ↓
         Firestore usage.해당항목 += 1 (원자적 increment)
          ↓
         결과 반환
```

### 3-4. 플랜 한도 상수 파일 (`lib/plans.ts`)

```typescript
export const PLAN_LIMITS = {
  guest:    { summarize: 3,   docent: 0,  blog: 0,  ... }, // 평생 한도
  free:     { summarize: 20,  docent: 30, blog: 3,  ... }, // 월 한도
  standard: { summarize: 150, docent: -1, blog: 30, ... }, // -1 = 무제한
  premium:  { summarize: 900, docent: -1, blog: -1, ... },
  teacher:  { summarize: -1,  docent: -1, blog: -1, ... },
}

// 교사 플랜 학생 한도 (일별)
export const STUDENT_DAILY_LIMITS = {
  summarize: 3,
  docent: 3,
}
```

### 3-5. 월 초기화 방식

- 별도 Cron 서버 없이 **API 호출 시점에 체크**
- `resetAt` < 오늘 → 그 자리에서 usage 초기화
- Vercel 서버리스 환경에 최적화된 방식

## 4. 결제 연동 (🔲 미확정)

### 후보
- **토스페이먼츠** — 국내 1위, 문서 좋음, 정기결제 지원
- **포트원(아임포트)** — 멀티 PG 지원, 카카오페이·네이버페이 통합

### 결제 흐름
```
유저 플랜 선택
  → 결제 위젯 오픈 (토스페이먼츠 SDK)
  → 결제 완료
  → 결제 웹훅 수신 (/api/webhook/payment)
  → Firestore users/{uid} 업데이트:
      plan = 'standard' | 'premium' | 'teacher'
      planExpiry = 오늘 + 30일
  → 유저에게 성공 화면 표시
```

### 정기결제 (구독)
- 토스페이먼츠 자동결제(빌링키) 사용
- 매월 planExpiry 전날 자동 청구
- 실패 시 3일간 재시도 → 실패하면 free로 다운그레이드

## 5. 프론트엔드 처리

### 한도 초과 시 UX
- API에서 `{ error: 'LIMIT_EXCEEDED', plan: 'free', feature: 'summarize' }` 반환
- 클라이언트에서 **업그레이드 모달** 자동 표시
- "이번 달 요약 20회를 모두 사용했어요. 스탠다드로 업그레이드하면 150회 이용 가능합니다."

### 사용량 표시
- 마이페이지 프로필 카드에 이번 달 사용량 바 표시
- ex) `요약 12 / 20회`
