# 세션 시작 시 필독 (MANDATORY)

새 대화를 시작할 때 반드시 아래 두 파일을 먼저 읽어라.
코드 작업 전에 이 파일들을 읽지 않으면 컨텍스트 없이 작업하는 것이다.

1. `HISTORY.md` — 지금까지 해결한 이슈·기능 히스토리. 같은 실수 반복 방지.
2. `TODO.md` — 남은 작업 목록과 우선순위. 다음에 뭘 해야 하는지 파악.
3. `JAPAN.md` — 일본 런칭 플랜. 코드 작업 시 일본어 다국어 확장을 염두에 두고 설계할 것.
4. `SHARE.md` — 공유하기 기능 확장 플랜 (카카오톡→Slack→Notion 순서).

읽은 후 사용자에게 현재 TODO 상태를 한 줄로 요약해서 알려줄 것.
